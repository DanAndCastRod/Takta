Este documento resume los hallazgos clave, activos identificados y decisiones estratégicas discutidas durante la reunión, para guiar el desarrollo de **Takta**.

## 1. Estrategia Dual: Open Source vs Enterprise

Se confirma la estrategia de dividir el proyecto en dos vertientes claras:

| Característica | **Takta Community (Open Source)** | **Takta Enterprise (Grupo Bios)** |
| :--- | :--- | :--- |
| **Audiencia** | Ingenieros industriales, PyMEs, Consultores | Plantas de Operadora Avícola / Grupo Bios |
| **Frontend** | **TailwindCSS** (Diseño moderno, agnóstico) | **Bios Design System** (Corporativo) |
| **Infraestructura** | Ejecución directa (No Docker inicial), SQLite/Postgres | Windows Server, IIS, SQL Server, Azure AD |
| **Integraciones** | Genéricas / CSV | SAP/Siesa, Mantum (Activos), Directorio Activo |

## 2. Funcionalidades Clave Identificadas

### A. Ingeniería de Métodos y Tiempos (Core)
*   **Referencia Metodológica:** Se cita explícitamente el libro **"Ingeniería Industrial: Métodos, Estándares y Diseño del Trabajo" de Nievel** (Ed. 13 o superior) como la "biblia" para los formatos y cálculos.
*   **Cronómetro Digital Avanzado:**
    *   No solo mide tiempo, también debe permitir **conteo de unidades/ítems** (inspirado en un metrónomo y cronómetros de mano).
    *   Interfaz *touch-first* para tablets.
*   **Cálculo de Capacidad (Roll-up):** Lógica para agregar capacidades desde el nivel más bajo (puesto de trabajo/máquina) hacia arriba (línea, área, planta).

### B. Gestión de Activos y Visualización de Planta
*   **Integración Mantum:** Para la versión Enterprise, los activos se sincronizan con Mantum. Uso de **Códigos QR** en máquinas para acceder a la "Hoja de Vida Digital" del activo.
*   **Mapas de Planta (Draw.io / SVG):**
    *   Se dispone de un plano de **Planta Pereira** en formato `draw.io` (XML/SVG).
    *   **Visión:** Cargar estos diagramas en Takta y usarlos como base interactiva.
    *   **Capas de Información (Layers):** Sobreponer "mapas de calor" al plano para visualizar:
        *   Estado de estándares (Semaforización: desactualizado/ok).
        *   Accidentalidad (zonas de riesgo).
        *   Cuellos de botella o flujo de producción.

### C. Ejecución y Bitácora (El "Waze" de la Producción)
*   **Bitácora de Novedades:** Registro de eventos en tiempo real (inicios, paradas, cambios).
*   **Dictado por Voz (Voice-to-Text):** Funcionalidad crítica para mantenimiento. Al atender una máquina, el técnico debe poder dictar el diagnóstico en lugar de escribir en una tablet incómoda.
*   **Contexto Automático:** Si un usuario logueado como "Supervisor Sellado" reporta una novedad, el sistema debe asumir automáticamente el área "Sellado".

### D. Mejora Continua (Kaizen Loop)
*   **Hallazgos y Acciones:** Sistema tipo Kanban para gestionar acciones correctivas que no se pierdan en correos.
*   **Auditorías 5S:** Listas de chequeo configurables.

## 4. Notas Técnicas Adicionales

*   **Editor de Documentos:** Se reitera el uso de un editor tipo "Notion" (Editor.js) que soporte **Mermaid** para diagramas de flujo integrados en los procedimientos.
*   **Tracking Físico:** Se aclaró que *no* se busca un tracking tipo GPS de cada pollo en tiempo real (por ahora), sino un tracking de **eventos y flujo lógico** entre procesos (Hoja de Ruta / Carta de Navegación).
